# 文書2: 表層的デザイン（UI/UX方針）

## 1. 全体方針
対象ユーザーが小中学生であることを最優先に考慮する。学習への心理的抵抗感を減らし、楽しく継続的に利用できるような、シンプルかつ視覚的に達成感を得られるデザインを採用する。

## 2. レイアウトとナビゲーション
* **基本構造:** 画面左側に主要機能（日報、相談、ARCHIVEなど）へアクセスするグローバルナビゲーション（サイドバー）を常時配置する。
* **コンテンツ領域:** メインコンテンツはナビゲーションの右側に表示する。ARCHIVEページなど、情報量が多い場合はタブ切り替えを採用する。
* **レスポンシブ:** PCとスマートフォンの両方の画面サイズで最適に表示されるレイアウトとする。

## 3. 色彩とタイポグラフィ
* **基調色:** 安心感や学習への集中を促す水色（ライトブルー）や白を背景の基調とする。
* **アクセントカラー:** カレンダーの活動日、バッジ、継続日数など、利用者の達成感に関わる要素には、オレンジ、金、銀、銅などの暖色系やメタリックカラーを使用し、視認性とモチベーション向上を図る。
* **フォント:** 読みやすく、親しみやすいゴシック体を基本とする。数字（継続日数など）は視認性の高いフォントで大きく表示する。

## 4. 主要コンポーネントのデザイン

### 4.1. チュートリアル（クエスト）画面
* **誘導:** 初回利用時にモーダルウィンドウや専用画面で表示する。
* **視覚的サポート:** 操作対象（例：「日報ボタン」）を矢印やハイライトで明示する。ガイド役のキャラクター配置も検討する。
* **進捗表示:** クエストのリスト（達成済み/未達成）やプログレスバーを表示し、チュートリアルの進捗が分かるようにする。
* **スキップ:** チュートリアルを途中で終了する「スキップ」ボタンを配置する。

### 4.2. 学習日報ページ
* **インターフェース:** 過去の日報とメンターからのフィードバックが時系列で一覧できるタイムライン形式を採用する。
* **入力フォーム:** 日付、教科、学習内容、コメントなどを簡潔に入力できるフォームを配置する。

### 4.3. 相談（チャット）ページ
* **インターフェース:** メンターとのやり取りは、一般的なチャットアプリケーション（例：LINE）のように、吹き出し形式で見やすく表示する。

### 4.4. ARCHIVE（実績）ページ
* **ヘッダー:** ページタイトル「ARCHIVE」を中央上部に配置する。
* **タブ:** 「カレンダー」タブと「スキル（バッジ）」タブを設け、利用者が表示を切り替えられるようにする。
* **カレンダータブ:**
    * **継続日数:** 「連続ログイン XXX 日」のように、成果を最も目立つ位置に大きく表示する。
    * **マンスリー表示:** 複数月（例：当月、前月、前々月）のカレンダーを表示し、活動日（ログイン、日報作成）を色やマークで強調する。
    * **ウィークリー表示:** 直近1週間の活動（S M T W T F S）を別途表示する。
* **スキル（バッジ）タブ:**
    * **バッジリスト:** 獲得したバッジ（金・銀・銅などのメダル形状）をグリッド形式で一覧表示する。
    * **説明欄:** 各バッジの名称や獲得条件（例：「まじめさ」「継続日数」）を表示するスペースを設ける。

### 4.5. アンケート画面
* **レイアウト:** 質問文と回答選択肢（または入力欄）が明確に区別できるように、十分な余白を持って配置する。
* **操作性:** 選択肢のボタンやチェックボックスは、押し間違えを防ぐため、十分な大きさと間隔を確保する。
* **完了:** 回答送信後、「ご協力ありがとうございました」などの完了メッセージを明確に表示する。
