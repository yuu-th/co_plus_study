# 003: CSS Modules採用

## 日付
2025-10-01

## ステータス
採用

## 決定
スタイリングはCSS Modules + CSS変数を使用する

## 背景
コンポーネントスコープのスタイリングと、グローバルなデザイントークン管理を両立したい。

## 理由
- **スコープ分離**: クラス名の衝突を自動回避
- **標準技術**: 追加ライブラリ不要、Vite標準サポート
- **CSS変数との相性**: グローバル変数を各モジュールから参照可能
- **学習コスト低**: 通常のCSSとほぼ同じ

## 検討した代替案

| 選択肢 | 長所 | 短所 | 判断 |
|--------|------|------|------|
| Tailwind CSS | 高速開発 | クラス名肥大、学習コスト | 不採用 |
| styled-components | JS内完結 | ランタイムコスト、依存追加 | 不採用 |
| Emotion | 柔軟 | 同上 | 不採用 |
| Sass/SCSS | 機能豊富 | CSS変数で十分 | 不採用 |

## 規約
- ファイル名: `ComponentName.module.css`
- クラス名: camelCase（BEM不使用）
- 変数参照: `var(--color-xxx)` 形式

## 影響
- グローバルスタイルは `shared/styles/` に最小限
- カラー・スペーシング等は CSS変数で一元管理
