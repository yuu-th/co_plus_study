# バックログ管理

## 概要

このディレクトリでタスク（Issue相当）を管理します。
外部ツールに依存せず、ファイルベースで可視化・追跡します。

## ディレクトリ

| ディレクトリ | 説明 | 制限 |
|-------------|------|------|
| `todo/` | 未着手タスク | なし |
| `doing/` | 進行中タスク | **最大3個** |
| `done/` | 完了タスク | 削除しない |

## タスクファイルのフォーマット

```markdown
# タスクタイトル

## 概要
{1-2文で説明}

## 参照仕様
→ specs/features/xxx.md

## チェックリスト
- [ ] サブタスク1
- [ ] サブタスク2

## メモ
{実装時の気づき、注意点}

## 履歴
- YYYY-MM-DD: 作成
- YYYY-MM-DD: 完了
```

## ワークフロー

```
1. タスク作成
   → todo/ にファイル作成

2. 着手
   → doing/ に移動
   → doing/ は最大3個まで（WIP制限）

3. 完了
   → ファイル内の履歴に完了日を追記
   → done/ に移動

4. 振り返り
   → done/ を見て完了タスクを確認
```

## 命名規則

- ファイル名: `{機能}-{内容}.md`（ケバブケース）
- 例: `diary-reaction-button.md`, `mentor-dashboard.md`

## 優先度

ファイル名の先頭に優先度を付けることも可能:
- `p1-緊急タスク.md`
- `p2-重要タスク.md`
- `p3-通常タスク.md`
