## 1. プロジェクト概要

### 1.1 システム名
**CO+ Study**（コープラス スタディ）

### 1.2 一言で言うと
学習意欲のある小中学生を、高専生メンターが支援する学習記録・相談Webアプリ

### 1.3 解決する課題

| 課題 | 解決アプローチ |
|------|----------------|
| 学力不足 | 学習の記録・可視化で継続を促進 |
| 教育格差 | 高専生メンターによる無料支援 |
| 視野の狭さ | 高専生との交流で将来の選択肢を広げる |
| やる気の空回り | 「やる気」を「実際の学び」に転換する仕組み |

### 1.4 対象ユーザー

| ロール | 対象 | 主な利用機能 |
|--------|------|--------------|
| 利用者 | 学びに興味のある小中学生 | 日報作成、相談、ARCHIVE閲覧 |
| メンター | 高専生 | 日報へのフィードバック、相談対応 |
| 管理者 | 運営者 | メンター管理、教科・単元管理 |

## 3. デザイン哲学

### 3.1 基本原則

**対象ユーザーが小中学生であることを最優先に考慮する。**

| 原則 | 具体的な方針 |
|------|-------------|
| シンプルさ | 機能を絞り、迷わないUI |
| 達成感の可視化 | カレンダー、バッジ、継続日数で成果を実感 |
| 心理的ハードル低減 | 学習への抵抗感を減らすカジュアルなトーン |
| 楽しさ | ゲーミフィケーション要素（クエスト、バッジ） |

### 3.2 レイアウト方針

```
┌─────────────────────────────────────────┐
│  Header（ロゴ、ユーザー情報）            │
├────────┬────────────────────────────────┤
│        │                                │
│ Side   │   Main Content                 │
│ bar    │   （メインコンテンツ領域）       │
│        │                                │
│ ・日報  │                                │
│ ・相談  │                                │
│ ・実績  │                                │
│  ...   │                                │
│        │                                │
└────────┴────────────────────────────────┘
```

- **サイドバー**: 主要機能へのナビゲーション（常時表示）
- **メインコンテンツ**: 選択した機能の内容
- **レスポンシブ**: モバイルではサイドバーが折りたたみ

### 3.3 色彩哲学

| 用途 | 色 | 意図 |
|------|-----|------|
| 基調色 | 水色（ライトブルー）、白 | 安心感、学習への集中 |
| アクセント | オレンジ、金・銀・銅 | 達成感、モチベーション向上 |
| 背景 | 薄いグレー系 | 目に優しく、長時間利用可能 |

→ 具体的なカラーコード: `shared/colors.md`

### 3.4 タイポグラフィ哲学

| 要素 | 方針 |
|------|------|
| 本文 | 読みやすいゴシック体、親しみやすさ |
| 数字（継続日数等） | 大きく、視認性の高いフォント |
| 見出し | 明確な階層、迷わない構造 |

→ 具体的なフォント設定: `shared/typography.md`

## 4. 技術スタック

### 4.1 現フェーズ

| 項目 | 内容 |
|------|------|
| フェーズ | バックエンド連携準備中 |
| バックエンド | Supabase（PostgreSQL + Auth + Realtime） |
| 認証方式 | 匿名認証 → OAuth連携（段階的） |

> 詳細: `project/decisions/005-backend-integration-preparation.md`

### 4.2 技術スタック

| レイヤー | 技術 |
|----------|------|
| フレームワーク | React 18 |
| 言語 | TypeScript (Strict) |
| ビルド | Vite |
| スタイリング | CSS Modules + CSS変数 |
| ルーティング | React Router v7 |
| データ取得 | React Query + Supabase Client |
| バックエンド | Supabase (PostgreSQL + RLS) |

### 4.3 禁止事項

| 禁止 | 理由 |
|------|------|
| `any` 型 | 型安全性の確保 |
| `React.FC` | 不要な型推論の問題 |
| CSS値のハードコード | CSS変数で一元管理 |
| 外部UIライブラリ | 学習目的、依存を減らす |
| styled-components / Emotion / Tailwind | CSS Modulesで統一 |
